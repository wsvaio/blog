## 关于本站

不知是因为程序员健忘，还是健忘的是我。

经常遇到一些似曾相识，却又无从想起的问题。

于是便有了此站，将那些问题无情地记录下来，未来有需要的时候再认真地扒拉出来……

这不仅是一个博客，更是我的数字花园、知识库和成长轨迹。这里记录着从前端到全栈的技术探索，从理论到实践的项目经验，以及从问题到解决方案的思考过程。

### 相关技术

想要完全独立搭建一个网站，只需要三步：

1. 设计
2. 开发
3. 部署

#### 项目设计

参考了很多博客主题，包括但不限：[butterfly](https://github.com/jerryc127/hexo-theme-butterfly)、[sakura](https://github.com/mashirozx/Sakura)、[cuteen](https://blog.zwying.com/)、[volantis](https://github.com/volantis-x/hexo-theme-volantis)……

#### 前端开发

身为一名前端开发工程师，这相当于本职工作了，不同的是，这次是为自己打工……

主要使用 **Nuxt 3** 实现 **SSR** 服务端渲染，**Pinia** 做状态管理，**UnoCSS** 原子化 CSS，**VueUse** 组合式工具库。

#### 核心技术栈

- **框架**: Nuxt 3 (Vue 3 + Vite)
- **语言**: TypeScript
- **样式**: UnoCSS + Less
- **状态管理**: Pinia
- **工具库**: VueUse
- **代码质量**: ESLint + Stylelint + Prettier + Oxlint

#### 后端开发

不会点后端，都不好意思说自己是前端（bushi）。

基于 **Nitro** 构建高性能 API 服务，**Prisma ORM** 管理数据库，**PostgreSQL** 作为主数据库，使用 **Bun** 运行时获得更快的启动速度和更低的内存占用。

#### 后端技术栈

- **运行时**: Bun (替代 Node.js)
- **API 框架**: Nitro (Nuxt 内置)
- **ORM**: Prisma
- **数据库**: PostgreSQL
- **认证**: JsonWebToken (JWT)
- **邮件**: Nodemailer
- **工具库**: @wsvaio/utils, @wsvaio/api

#### 运维部署

看似遥不可及，实际一点都不接近。目的很存粹，能跑就行（笑）。

全站容器化部署，所有服务打包成 Docker 镜像，通过 Kubernetes 编排管理，使用 GitHub Actions 实现完整的 CI/CD 流程。从代码提交到自动构建、测试、部署，全程无需人工干预。

#### DevOps 工具链

- **版本控制**: Git + GitHub
- **CI/CD**: GitHub Actions
- **容器化**: Docker
- **编排**: Kubernetes (k8s)
- **反向代理**: Nginx
- **系统**: Ubuntu Server
- **包管理**: pnpm
